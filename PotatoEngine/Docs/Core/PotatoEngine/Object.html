Object

Objects are the second most basic part of a PotatoEngine project, after Components. An Object may contain any number of Components.


std::string name

name is a string used as an identifier. It can be used to find an Object.


long int uid

uid is a unique identifier for each Object. It is set on the creation of an Object


std::vector<std::unique_ptr<Component>> components

components contains pointers to all child Components.


T* Add_Component(Args&&... args)


T* Get_Component()


void Start()

Start is called when a Scene is created or activated for the first time, or when this Object is first created. Start calls its counterparts in all child Components.


            // Implementation in Object.hpp
            void Start() {
                for (auto& component : components) {
                    component.Start();
                }
            }

void Update(double delta_time)

Update is called as often as the engine can, as long as it is less than 1ms. Update calls its counterparts in child Components.


            // Implementation in Object.hpp
            void Update(double delta_time) {
                for (auto& component : components) {
                    component.Update(delta_time);
                }
            }

void Fixed_Update(double delta_time)

Fixed_Update is called as close to a fixed rate as possible (by default, every 1/60th of a second). It calls its counterparts in child Compoments.


            // Implementation in Object.hpp
            void Fixed_Update(double delta_time) {
                for (auto& component : components) {
                    component.Fixed_Update(delta_time);
                }
            }
