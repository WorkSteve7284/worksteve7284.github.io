PotatoEngine Docs

Welcome to the PotatoEngine Docs! This website contains all of the details on how PotatoEngine works.


Styling

The various pages on this site use font and background colors to indicate what some words mean.

Classes: this color indicates that something is a class.

Functions: this color indicates that something is a function. Depending on the use case, this may or may not include parenthises ().

variables: this color indicates that something is a variable. The type of a variable will usually have been indicated previously, but can also be viewed by hovering over it.


variable, Class, and Function naming conventions

As you may know, there is no correct way to write your code. In languages like C++, you could have all of your code on the same line (but shouldn't). As a result, every programmer has their own way of writing code. All of the official PotatoEngine libraries use snake_case for variables and Upper_Snake_Case for Classes and Functions

File names, however, are in PascalCase/UpperCamelCase.


Tips and Warnings

Throughout the docs, there are tips and warnings. Tips are used to help with good coding practice and ease of use, while warnings are used for telling you to NEVER do something
